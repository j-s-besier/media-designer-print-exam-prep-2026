# Aufgabenformen AP Sommer 2026

Mediengestalter Digital und Print, Fachrichtung Printmedien  
Neue Verordnung 2023  
Schriftliche Prüfung: 6. Mai 2026

Diese Übersicht beschreibt, in welcher Form Aufgaben zu den veröffentlichten Themen wahrscheinlich gestellt werden. Grundlage sind die ZFA-Themen Sommer 2026, die ZFA-Beispielprüfung AP 2025 nach neuer Verordnung und ergänzend Aufgabenmuster aus der alten Verordnung.

Legende:

- G = in neuer VO/ZFA-Beispielprüfung gefunden
- A = aus alter VO-Beispielprüfung bekannt
- V = fachlich plausibel vermutet

| Themenbereich Printmedien Sommer 2026 | Gefundene oder vermutete Aufgabenstellungsweisen |
|---|---|
| Corporate-Design-Beratung | V: kurze Textantworten, Beratungsfragen formulieren, CD/CI/Styleguide erklären, Mängel in CD-Anwendung benennen, Tabelle mit CD-Elementen ausfüllen |
| Zielgruppenanalyse | A/G/V: Merkmale nennen, Zielgruppe aus Situation ableiten, Tabelle Zielgruppenmerkmal -> Gestaltung, kurze Begründung, Persona/Steckbrief |
| Logo-Entwurf | G/A: Logo bewerten, Kriterien nennen, Ja/Nein-Entscheidung mit Begründung, semantische Wirkung beurteilen, technische Kriterien in Tabelle ergänzen |
| Webseiten-Gestaltung | A/V: Fachbegriffe erklären, Struktur/Navigationskonzept beschreiben, Usability/Barrierefreiheit beurteilen, Tabelle mit Anforderungen, ggf. kleine Skizze/Wireframe |
| Flyer-Gestaltung | V/A: Gestaltungsanalyse, Informationshierarchie erstellen, Format/Falz/Blickführung begründen, Verbesserungsvorschläge formulieren |
| Schutzrechte | G: Schutzrechte nennen und erklären, Fallbeispiel beurteilen, Schutzumfang/Zustandekommen beschreiben, Ja/Nein-Bewertung mit Begründung |
| Korrekturen durchführen | G/A: Fehler erkennen, Problem + Korrekturmöglichkeit in Tabelle eintragen, Preflight vs. visuelle Kontrolle ankreuzen, Korrekturzeichen/Korrekturablauf beschreiben |
| Protokolle | V: Protokollarten unterscheiden, Protokollstruktur ergänzen, fehlerhafte Protokollnotizen korrigieren, Beschlüsse/To-dos formulieren |
| Gestaltungsraster | G/A: Satzspiegel/Spalten berechnen, Rasterbestandteile erklären, Registerhaltigkeit begründen, Raster skizzieren, Maße eintragen |
| Gestaltungsgrundsätze anwenden | G/A: Gestaltung nach Kriterien analysieren, Verbesserungsvorschläge, aktive/leise Auszeichnung zuordnen, Plakat/Flyer/Layout beurteilen |
| Visuelle Zeichen | A/V: Symbol/Icon/Piktogramm erklären, Zeichenwirkung beurteilen, Semantik analysieren, Zeichen einer Funktion zuordnen, ggf. einfache Skizze |
| Gestaltung beurteilen | G/A: Entwurf bewerten, Kriterien nennen, Wirkung begründen, Ja/Nein-Entscheidung, Tabelle Kriterium -> Bewertung -> Verbesserung |
| Feedback | V: gutes/schlechtes Feedback unterscheiden, Feedbackregeln nennen, Kundenfeedback in Arbeitsaufträge umwandeln, Kritik fachlich formulieren |
| HTML/CSS-Syntax | A/V: Codefehler erkennen/korrigieren, Selektoren/Tags erklären, Zuordnung Tag/Funktion, kurze Code-Ergänzung, Ankreuz-/Tabellenaufgabe |
| Druckverfahren | G/A/V: Verfahren Produkten zuordnen, Vor-/Nachteile nennen, Einschränkungen für Farbe/Material erklären, Tabelle Produkt -> Druckverfahren -> Begründung |
| KI | V: Einsatzfelder nennen, Risiken erklären, Fallbeispiel beurteilen, Workflow beschreiben, Recht/Datenschutz/Qualität bewerten |
| Bildbeurteilung | G/A: Auflösung/Datenmenge/Farbtiefe berechnen, Dateiformate vergleichen, Bildeigenschaften zuordnen, Bildqualität beurteilen, Korrekturmaßnahmen nennen |
| Medienneutrale Daten | G/A: Begriff erklären, Vorteile begründen, Begriffe zu Merkmalen zuordnen, Workflow beschreiben, Tabelle ausfüllen |
| Englischsprachige Informationsquellen | G/V: englische Fachbegriffe übersetzen, Prozessschritte aus englischer Quelle übertragen, Tabelle vervollständigen, technische Angaben interpretieren |
| Arbeitsdokumentation | V/A: Dokumentationsstruktur erstellen, Arbeitsschritte chronologisch ordnen, Dateinamen/Versionen bewerten, Protokoll- oder Checklistenform |
| Produktionsdaten aufbereiten | G/A: Preflight-Checkliste, Fehler erkennen, PDF/X/Beschnitt/Farbprofil/Schriften erklären, Tabelle Fehler -> Auswirkung -> Korrektur |
| Druckbogen-Berechnung | G/A: Nutzen berechnen, Druckbogen skizzieren, Maße eintragen, Laufrichtung/Falz/Schnitt ankreuzen, Ausschießschema erstellen |
| Farbmanagement | G/A: Farbprofile zuordnen, Rendering Intents erklären, RGB/CMYK-Konvertierung beurteilen, Ankreuzaufgaben, Farbwiedergabe begründen |
| Automatisierung | G/V: automatisierte Bildoptimierung erklären, automatisierbare Tätigkeiten nennen, Grenzen/Risiken beurteilen, Workflow oder Checkliste beschreiben |

## Prüfungsstil Nach Neuer Verordnung

- Am häufigsten: kurze Textantworten, Tabellen, Zuordnungen und Begründungen.
- Sehr wahrscheinlich bei Printmedien: Berechnung, Skizze, Druckbogen-/Raster-/Farbmanagement-Aufgaben.
- Möglich, aber seltener: klassisches Multiple Choice.
- Häufiger als Multiple Choice: Ankreuzen in Tabellen.
- Klassischer Lückentext ist eher unwahrscheinlich; eher werden Tabellenfelder ergänzt.
- Bei HTML/CSS kann eine Code-Korrektur oder Begriffserklärung kommen.
